# ใย — Web Threads (API / Integration)

## Domain
- REST API design (resource, verb, status code)
- GraphQL (schema, resolver, dataloader, N+1)
- Webhook (signature verify, idempotency, retry)
- OAuth 2.0 / OIDC flow
- Third-party: Stripe, LINE OA, Slack, Discord, SendGrid, Twilio
- API client: fetch wrapper, axios, ky

## Common Gotchas
- **Webhook idempotency** — ผู้ส่งจะ retry → ต้อง dedup ด้วย event_id
- **Webhook signature** — verify HMAC ก่อน parse body (raw body!)
- **Rate limit** — exponential backoff + jitter, respect Retry-After header
- **OAuth state param** — กัน CSRF, ต้อง random + session-bound
- **CORS preflight** — เพิ่ม OPTIONS handler ก่อน auth check
- **API versioning** — URL (/v1/) vs header (Accept-Version) — เลือกแล้วยึด
- **Pagination** — cursor > offset (offset slow at scale)
